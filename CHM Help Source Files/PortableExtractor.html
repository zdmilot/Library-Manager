Portable Package Extractor
Library Manager » Portable Package Extractor

Overview

Library Manager ships a portable extraction tool (hxlibpkg-extract.exe) that can unpack .hxlibpkg and .hxlibarch files without requiring the full Library Manager application. This is useful in environments where the main application cannot be installed, for automated workflows, or when a user simply needs to inspect or retrieve files from a library package.

The tool is a standalone Windows executable with no external dependencies — it runs on any Windows 10 or later machine using only the built-in .NET Framework 4.x.

Note: The portable extractor only extracts package contents to a local directory. It does not install libraries into the VENUS environment, register COM objects, or modify the Library Manager database. For full library installation, use the Library Manager GUI or CLI.

Location

When Library Manager is installed, the extractor tool is located at:

C:\Program Files (x86)\Library Manager\tools\hxlibpkg-extract\hxlibpkg-extract.exe

The tool can also be copied to any other location and run independently. It has no dependencies beyond the Windows .NET Framework.

A Node.js version (hxlibpkg-extract.js) is also included in the same directory for cross-platform use or integration into Node.js-based workflows.

Usage

hxlibpkg-extract.exe <package> [options]

Options

FlagDescription
--out <dir>Specify output directory. Default: .\<library_name>\
--listList package contents without extracting
--manifestPrint the manifest.json and exit
--helpShow help message

Examples

Extract a Package

Extract a single library package to a directory named after the library:

hxlibpkg-extract.exe MyLibrary.hxlibpkg

This creates a MyLibrary\ directory in the current folder containing all extracted files.

Extract to a Specific Directory

hxlibpkg-extract.exe MyLibrary.hxlibpkg --out C:\Users\me\Desktop\output

List Package Contents

View what’s inside a package without extracting:

hxlibpkg-extract.exe MyLibrary.hxlibpkg --list

Example output:

Package: MyLibrary v1.0.0
8 entries:
  library/MyLibrary.hsl  (19742 bytes)
  library/MyLibrary.bmp  (3128 bytes)
  library/MyLibrary.ico  (9062 bytes)
  demo_methods/MyDemo.med  (5540 bytes)
  help_files/MyLibrary.chm  (46430 bytes)
  manifest.json  (5361 bytes)

View Package Manifest

Print the JSON manifest (metadata) of a package:

hxlibpkg-extract.exe MyLibrary.hxlibpkg --manifest

Extract a Multi-Library Archive

Archives (.hxlibarch) contain multiple packages. Each package is extracted into its own subdirectory:

hxlibpkg-extract.exe MyArchive.hxlibarch --out C:\output

This produces:

C:\output\
  LibraryA\
    manifest.json
    library\
    demo_methods\
  LibraryB\
    manifest.json
    library\
    ...

Output Structure

Extracted packages follow a standardized directory layout:

DirectoryContents
manifest.jsonPackage metadata (library name, version, author, etc.)
library\Library files: .hsl, .dll, .bmp, .ico, etc.
demo_methods\Demo method files (.med, etc.)
help\Help files (.chm, etc.)
installer\Embedded installer executables (if present)
icon\Package icon image
bin\Binary files for Hamilton\Bin (if present)

Integrity Verification

The extractor performs full HMAC-SHA256 integrity verification before extracting any data. If a package has been corrupted or tampered with, extraction is refused entirely with a clear error message:

Error: Package integrity check failed: the file has been corrupted or tampered with.

This is the same fail-closed integrity verification used by the full Library Manager application. See Binary Container Format for details.

Security

  • HMAC-SHA256 integrity: Every package is verified before extraction. Tampered or corrupted files are rejected.
  • Path traversal protection: The extractor refuses to write files outside the target directory, preventing malicious packages from overwriting system files.
  • Timing-safe comparison: HMAC comparison uses constant-time comparison to prevent timing attacks.

Differences from Full Library Manager

The portable extractor is a subset of the full Library Manager. It intentionally does not:

  • Install libraries into the VENUS library directory
  • Register COM DLLs
  • Run embedded installers
  • Modify the Library Manager database
  • Verify Ed25519 code signatures (use the full app or CLI verify-package command)
  • Require administrator privileges

When to Use the Portable Extractor

  • You need to inspect the contents of a .hxlibpkg file without installing it
  • Library Manager cannot be installed on the target machine
  • You need to extract library files for use in an automated build or CI/CD pipeline
  • You want to retrieve specific files (e.g., HSL source, demo methods) from a package
  • A colleague shared a package file and you need the library files without the full app

Building from Source

The extractor source code (HxlibpkgExtract.cs) is included in the tools\hxlibpkg-extract\ directory. To rebuild:

cd tools\hxlibpkg-extract
build.bat

Or manually:

C:\Windows\Microsoft.NET\Framework\v4.0.30319\csc.exe /out:hxlibpkg-extract.exe /target:exe /optimize+ /r:System.IO.Compression.dll /r:System.IO.Compression.FileSystem.dll HxlibpkgExtract.cs
