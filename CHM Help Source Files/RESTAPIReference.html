REST API Reference
Library Manager for Venus 6 » REST API Reference

Overview

Library Manager for Venus 6 ships a fully-featured REST API server (rest-api.js) with OpenAPI 3.0 (Swagger) documentation. The REST API provides complete 1:1 parity with the CLI commands and is also the transport used by the .NET COM object.

Prerequisites

  • Node.js must be installed (or use the NW.js runtime from the application directory)
  • All Node.js dependencies (express, swagger-ui-express, multer, adm-zip, diskdb) are satisfied by the project's node_modules

Starting the Server

cd "C:\Program Files (x86)\Library Manager for Venus 6"
node rest-api.js

Command-Line Options

FlagDescriptionDefault
--port <number>HTTP port to listen on5555
--host <address>Bind address127.0.0.1 (localhost only)
--api-key <key>Require API key authentication via X-API-Key headerdisabled

Examples

REM Start on default port 5555
node rest-api.js

REM Start on custom port with API key
node rest-api.js --port 8080 --api-key my-secret-key

REM Listen on all interfaces (for network access)
node rest-api.js --host 0.0.0.0 --port 5555

Swagger UI

Once the server is running, open a browser to:

  • Swagger UI: http://localhost:5555/docs — Interactive API explorer with "Try it out" buttons
  • OpenAPI JSON: http://localhost:5555/api-docs — Raw OpenAPI 3.0 specification

The Swagger UI lets you test every endpoint interactively, view request/response schemas, and generate client code.

Authentication

If the server is started with --api-key, all /api/* requests must include the header:

X-API-Key: <your-key-here>

Requests without a valid key receive a 401 Unauthorized response.

Concurrency

Important: The underlying database (diskdb) uses flat JSON files without file-level locking. The REST API implements an internal mutex to serialize mutating operations (import, delete, rollback). Read operations (list, get, export) are safe to call concurrently.

Response Format

All JSON responses share a common envelope:

{
  "success": true,
  "data": { ... },
  "warnings": ["optional warning messages"]
}

Error responses:

{
  "success": false,
  "error": "Description of the error"
}

HTTP Status Codes

Quick Start Examples (cURL)

List all libraries

curl http://localhost:5555/api/libraries

Import a package

curl -X POST http://localhost:5555/api/libraries/import \
  -F "package=@MyLibrary.hxlibpkg"

Export a library

curl http://localhost:5555/api/libraries/MyLibrary/export \
  --output MyLibrary.hxlibpkg

Delete a library

curl -X DELETE http://localhost:5555/api/libraries/MyLibrary

Verify a package

curl -X POST http://localhost:5555/api/packages/verify \
  -F "package=@MyLibrary.hxlibpkg"

Quick Start Examples (PowerShell)

List all libraries

Invoke-RestMethod http://localhost:5555/api/libraries

Import a package

$form = @{ package = Get-Item ".\MyLibrary.hxlibpkg" }
Invoke-RestMethod -Uri http://localhost:5555/api/libraries/import `
  -Method POST -Form $form

Health check

Invoke-RestMethod http://localhost:5555/api/health

Security Considerations

Network Binding: By default the server only listens on 127.0.0.1 (localhost). If you bind to 0.0.0.0 with --host to allow remote access, you must also use --api-key to prevent unauthorized access to library management operations.

  • Package signing and integrity verification are enforced identically across CLI, REST, and COM
  • OEM author restrictions and publisher trust are checked on every import
  • The audit trail records all operations regardless of which interface is used
