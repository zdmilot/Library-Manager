REST API: POST /api/libraries/import-archive
Library Manager for Venus 6 » REST API Reference » Import Archive

Endpoint

POST /api/libraries/import-archive

Description

Imports a multi-library .hxlibarch archive file via multipart file upload. Equivalent to the CLI command import-archive. The archive contains one or more .hxlibpkg packages that are each extracted and imported individually.

This endpoint is serialized by an internal mutex to prevent concurrent database writes.

Parameters

Request Body (multipart/form-data)

FieldTypeRequiredDescription
archivefileRequiredThe .hxlibarch archive file to import (max 500 MB)
forcestringNo"true" to overwrite already-installed libraries
noGroupstringNo"true" to skip auto-assigning libraries to groups
noCachestringNo"true" to skip caching packages in the package store
authorPasswordstringNoPassword to authorize importing packages with restricted OEM author names

Response

Returns the standard envelope with data containing import results for each library in the archive:

  "success": true,
  "data": {

Examples

cURL

# Basic archive import
curl -X POST http://localhost:5555/api/libraries/import-archive \
  -F "archive=@AllLibraries.hxlibarch"

# Force overwrite all
curl -X POST http://localhost:5555/api/libraries/import-archive \
  -F "archive=@AllLibraries.hxlibarch" \
  -F "force=true"

PowerShell

# Basic archive import
$form = @{ archive = Get-Item ".\AllLibraries.hxlibarch" }
Invoke-RestMethod -Uri http://localhost:5555/api/libraries/import-archive `
  -Method POST -Form $form

# Force overwrite all
$form = @{
    archive = Get-Item ".\AllLibraries.hxlibarch"
    force   = "true"
}
Invoke-RestMethod -Uri http://localhost:5555/api/libraries/import-archive `
  -Method POST -Form $form

Error Conditions

  • No archive file uploaded → 400
  • Invalid archive file format → 400
  • One or more packages fail to import → partial results returned with per-library error details
