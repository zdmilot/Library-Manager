CLI: scan-markers
Library Manager » CLI Reference » scan-markers

Description

Scans a directory tree for .libmgr marker files and reports all managed libraries found. This is useful for discovering which libraries are installed in a given folder without relying on the Library Manager database. Typical use cases include recovery after a database loss, auditing a shared library folder, or verifying a fresh installation.

Usage

node cli.js scan-markers [--dir <path>] [--json]

Options

FlagRequiredDescription
--dir <path>NoDirectory to scan. Defaults to the configured Hamilton Library path (typically C:\Program Files (x86)\HAMILTON\Library)
--jsonNoPrint machine-readable JSON output instead of human-readable text

How It Works

  1. The scan root directory is resolved from --dir or the configured library path
  2. The root directory itself is checked for a .libmgr marker
  3. Each immediate subdirectory is checked for a .libmgr marker
  4. For each marker found, the embedded library metadata (name, version, author) is extracted and reported

The scan checks only one level of subdirectories, since libraries are installed directly under the Library root folder.

Examples

:: Scan the default library directory
node cli.js scan-markers

:: Scan a specific directory
node cli.js scan-markers --dir "D:\Hamilton\Library"

:: Machine-readable output for scripting
node cli.js scan-markers --json

JSON Output Format

When --json is specified, the output is a JSON array of marker objects, one per directory containing a marker:

[
  {
    "dirPath": "C:\\Program Files (x86)\\HAMILTON\\Library\\MyLibrary",
    "libraries": {
      "MyLibrary": {
        "version": "1.0.0",
        "author": "Jane Smith"
      }
    }
  }
]

Use Cases

  • Database recovery: If the Library Manager database is lost or corrupted, scan-markers can identify which libraries are installed on disk so they can be re-registered
  • Auditing: Verify which managed libraries exist in a shared or networked library folder
  • Migration: Before migrating to a new machine, scan the library folder to inventory installed libraries
