REST API: POST /api/libraries/:name/rollback
Library Manager for Venus 6 » REST API Reference » Rollback Library

Endpoint

POST /api/libraries/:name/rollback

Description

Reinstalls a previously cached version of a library from the package store. Equivalent to the CLI command rollback-lib. You can specify the target version by its version string or by its 1-based cache index.

This endpoint is serialized by an internal mutex to prevent concurrent database writes.

Parameters

Path Parameters

ParameterTypeRequiredDescription
namestringRequiredLibrary name

Request Body (application/json)

FieldTypeRequiredDescription
versionstringNo*Version string to roll back to (e.g. "1.0.0")
indexintegerNo*1-based cache index to roll back to
noGroupbooleanNoSkip auto-assigning the library to a group after rollback
authorPasswordstringNoOEM author password if the package has a restricted author

* Either version or index must be provided.

Response

Returns the standard envelope with data containing the rollback result:

Example Response

{
  "success": true,
  "data": {
    "libraryName": "MyLibrary",
    "rolledBackTo": "1.0.0",
    "previousVersion": "2.0.0",
    "filesExtracted": 8,
    "libInstallPath": "C:\\Program Files (x86)\\HAMILTON\\Library\\MyLibrary",
    "signatureStatus": "valid"
  }
}

Examples

cURL

# Rollback by version string
curl -X POST http://localhost:5555/api/libraries/MyLibrary/rollback \
  -H "Content-Type: application/json" \
  -d '{"version": "1.0.0"}'

# Rollback by cache index
curl -X POST http://localhost:5555/api/libraries/MyLibrary/rollback \
  -H "Content-Type: application/json" \
  -d '{"index": 1}'

PowerShell

# Rollback by version string
$body = @{ version = "1.0.0" } | ConvertTo-Json
Invoke-RestMethod -Uri http://localhost:5555/api/libraries/MyLibrary/rollback `
  -Method POST -ContentType "application/json" -Body $body

# Rollback by cache index
$body = @{ index = 1 } | ConvertTo-Json
Invoke-RestMethod -Uri http://localhost:5555/api/libraries/MyLibrary/rollback `
  -Method POST -ContentType "application/json" -Body $body

Error Conditions

  • Neither version nor index provided → 400
  • Specified version not found in cache → 400
  • Index out of range → 400
  • No cached versions available → 400
  • Restricted OEM author without authorPassword → 400
