CLI: trust-publisher
Library Manager for Venus 6 » CLI Reference » trust-publisher

Description

Adds or revokes trust for a publisher’s code signing certificate. When a publisher certificate is trusted, packages signed by that publisher show TRUSTED status on import and verification. When revoked, packages from that publisher show UNTRUSTED.

Usage

node cli.js trust-publisher --cert <cert.json> [--revoke]

Options

FlagRequiredDescription
--cert <path>RequiredPath to the publisher .cert.json file
--revokeOptionalRevoke trust instead of granting it

Certificate Validation

The certificate file is validated before being registered. The following checks are performed:

  • cert_format must be "hxlibpkg-publisher-cert/1.0"
  • publisher field must be present and non-empty
  • public_key must be a valid 32-byte Ed25519 key (base64-encoded)
  • fingerprint must match the SHA-256 hash of the public key
  • key_id must be present

If validation fails, the command prints the specific validation errors and exits.

Examples

:: Trust a publisher certificate
node cli.js trust-publisher --cert jane_smith.cert.json

:: Revoke a previously trusted certificate
node cli.js trust-publisher --cert jane_smith.cert.json --revoke

Publisher Registry

Trust state is stored in local/publisher_registry.json. Each publisher can have multiple certificates (e.g., for key rotation). Trust is managed per-certificate, not per-publisher — revoking one certificate does not affect other certificates from the same publisher.
